# Codexy

Codexy는 Codex에서 사용할 하네스와 루프 엔지니어링을 실험하고 정리하는 저장소입니다. 목표는 에이전트 실행 루프, 검증 하네스, 작업 증거 기록, 워크플로 자동화, 그리고 Codex 작업 품질을 높이는 작은 도구와 규칙을 한곳에서 관리하는 것입니다.

## 원칙

- Codex 작업은 재현 가능한 증거와 함께 남깁니다.
- 하네스, 루프, 검증 도구는 실제 작업 표면에서 관찰 가능한 결과를 우선합니다.
- 저장소 규칙은 에이전트가 바로 실행할 수 있도록 명령과 완료 조건을 분명히 적습니다.
- 작업 단위는 이슈 크기로 유지하고, `main`은 PR을 통해서만 갱신합니다.

## Git 워크플로

모든 기능, 문서, 정리, 설정 작업은 다음 흐름을 따릅니다.

1. 작업을 설명하는 GitHub 이슈를 먼저 만들거나 기존 이슈를 확인합니다.
2. `main`에서 직접 구현하지 않고, `eunsoogi/` 접두사의 토픽 브랜치를 별도 worktree에 만듭니다.
3. 커밋 제목은 Conventional Commit 형식을 사용합니다.
4. 브랜치를 푸시하고 이슈와 연결된 Pull Request를 엽니다.
5. 검증 결과와 남은 리스크를 PR 본문에 기록합니다.
6. PR은 squash merge로만 병합하고, 병합 후 원격 브랜치를 삭제합니다.
7. 병합 후 기본 worktree에서 `git pull --ff-only origin main`으로 `main`을 갱신합니다.

상세 규칙은 [AGENTS.md](AGENTS.md)와 [.agents/skills/git-workflow/SKILL.md](.agents/skills/git-workflow/SKILL.md)를 따릅니다.

## 저장소 정책

- 라이선스는 [MIT](LICENSE)입니다.
- GitHub 병합 방식은 squash merge만 허용합니다.
- PR 병합 후 브랜치 삭제를 켭니다.
- `main` 직접 수정은 금지하며 PR 병합으로만 반영합니다.

## 현재 상태

초기 저장소 규칙과 문서를 만드는 단계입니다. 코드, 패키지 구조, 테스트 명령은 실제 하네스와 루프 구현이 들어오면 이 README에 함께 추가합니다.
